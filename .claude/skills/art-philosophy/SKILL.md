# Digital Minimalism for Generative Art

A framework for creating systematic, non-representational generative art that produces specific objects through anti-compositional methods.

## Philosophy

This skill guides the creation of generative art based on three core principles:

1. **Specific Objects** - Create discrete digital entities, not demonstrations of algorithms
2. **Anti-Compositional** - Discover arrangements through systematic processes, not aesthetic design
3. **Thinking Without Thoughts** - Enable pure perception without symbolism or interpretation

## When to Use This Skill

Use when creating:
- Browser-based generative art (Canvas, SVG, WebGL)
- Systematic visual systems with clear constraints
- Time-based digital objects (videos, animations)
- Work influenced by minimalist art (Judd, Kelly, Knoebel, LeWitt)

Do NOT use for:
- Data visualization with specific meaning
- Representational or figurative work
- Work requiring symbolic interpretation
- Commercial graphics or branding

## Core Principles Explained

### 1. Specific Objects
Digital works must exist as discrete entities in their native medium, not representations of anything beyond themselves.

**In practice:**
- Each output is a complete, bounded thing
- Canvas dimensions are intentional and fixed
- Videos have defined duration and resolution
- The work can stand alone without series context
- Digital materiality is honest (pixels, code, compression)

**Avoid:**
- Infinite scrolling or unbounded generation
- Works that require explanation
- Pieces that only make sense in a series
- Mimicking physical materials (paint textures, paper grain)

### 2. Anti-Compositional
Arrangements emerge from systematic processes rather than aesthetic decision-making.

**In practice:**
- Use algorithmic constraints to discover compositions
- Randomness reveals possibilities within your system
- Outputs surprise you while remaining logically consistent
- The system can generate variations you wouldn't design manually
- "Found" arrangements feel inevitable given the constraints

**Avoid:**
- Hand-placing elements for aesthetic balance
- Cherry-picking outputs that look "good"
- Traditional design principles (rule of thirds, golden ratio)
- Manually tweaking results after generation

### 3. Thinking Without Thoughts
Works are encountered through pure perception, without interpretation or symbolic meaning.

**In practice:**
- Visual logic is immediately graspable
- No metaphor or narrative
- Form exists for structural relationships only
- Systematic process is visible but not the subject
- Explanation would diminish the encounter

**Avoid:**
- Representational imagery
- Symbolic color choices
- Narrative or conceptual frameworks
- Work requiring artist statements to understand

## The Algorithm Problem

**Warning:** The algorithm's aesthetic can easily become the work's aesthetic.

### Signs of Algorithm Dominance (Bad):
- Immediately recognizable as "noise art" or "particle system"
- Process is more interesting than results
- Looks like tutorials or other work using same technique
- Viewer sees the algorithm first, object second

### Signs of Proper Algorithm Use (Good):
- Systematic logic is clear but not the subject
- Can't immediately reverse-engineer the exact process
- Constraints push algorithm beyond natural tendencies
- Process serves vision rather than defining it

## System Development Process

### 1. Define Constraints
Start with tight, explicit constraints that limit but don't predetermine outcomes.

### 2. Let the System Surprise You
Run the system and observe what emerges. The goal is outputs you wouldn't have designed manually.

### 3. Evaluate Against Principles
Ask: Is this a specific object? Did the arrangement emerge from process? Can it be encountered without interpretation?

### 4. Adjust Constraints, Not Outputs
If results don't satisfy, change the system — not the individual output.
