Lesson 05 / WebGPU for generative art

The destination

The Clifford Torus

Everything you've built was groundwork for this. The engine is unchanged — ray, march, normal, shade. Only map gets stranger. The core idea: this torus lives on a sphere in 4D, and we project it into 3D. Rotating it in 4D is what makes it turn inside-out.

Three new functions to assemble. I'll port two for you (they show off WGSL patterns worth seeing); the third — pMod2 — is your final port.

1 · Inverse stereographic projection

This lifts a 3D point p up onto the unit 3-sphere in 4D, returning a vec4 and a scale factor k (needed later). In GLSL that second output is an out parameter — WGSL has none, so we return a struct:

GLSL 
vec4 inverseStereographic(vec3 p, out float k) {
    k = 2.0/(1.0+dot(p,p));
    return vec4(k*p, k-1.0);
}
WGSL 
struct Stereo { p4: vec4f, k: f32 };

fn inverseStereographic(p: vec3f) -> Stereo {
  let k = 2.0 / (1.0 + dot(p, p));
  return Stereo(vec4f(k * p, k - 1.0), k);
}
Pattern: out → struct Any GLSL function with extra out values becomes a WGSL function that returns a small struct. Cleaner than pointers when you're producing data rather than mutating the caller's.

2 · The 4D torus (meet select and atan2 for real)

The distance to the torus, plus surface UVs — another out. Two porting traps fire here at once:

GLSL 
float fTorus(vec4 p4, out vec2 uv) {
    float d1 = length(p4.xy) / length(p4.zw) - 1.;
    float d2 = length(p4.zw) / length(p4.xy) - 1.;
    float d = d1 < 0. ? -d1 : d2;
    d /= PI;
    uv = (vec2(atan(p4.y, p4.x), atan(p4.z, p4.w)) / PI) * .5 + .5;
    return d;
}
WGSL 
struct TorusHit { d: f32, uv: vec2f };

fn fTorus(p4: vec4f) -> TorusHit {
  let d1 = length(p4.xy) / length(p4.zw) - 1.0;
  let d2 = length(p4.zw) / length(p4.xy) - 1.0;
  var d  = select(d2, -d1, d1 < 0.0);   // d1 < 0. ? -d1 : d2   (false, true, cond!)
  d = d / PI;
  let uv = (vec2f(atan2(p4.y, p4.x), atan2(p4.z, p4.w)) / PI) * 0.5 + 0.5;
  return TorusHit(d, uv);
}
The two traps d1 < 0. ? -d1 : d2 → select(d2, -d1, d1 < 0.0) — false value first! · atan(y,x) → atan2(y, x). (PI is a module const PI = 3.14159265359;.)

Rotating in 4D — you can't point at a swizzle

map rotates planes of p4 with your pR: pR(p4.zy, …). But WGSL won't let you take a pointer to a swizzle like &p4.zy — so you extract, rotate, rebuild:

WGSL 
var zy = vec2f(p4.z, p4.y);
pR(&zy, u.time * -PI / 2.0);
p4 = vec4f(p4.x, zy.y, zy.x, p4.w);   // put z,y back (mind the order!)

This is the concrete reason the "return a value" route is often nicer than pointers — but here pR works fine, you just can't hand it a swizzle directly.

3 · fixDistance

The stereographic projection warps distances so the raymarcher would overshoot. This hand-tuned curve squashes them back to safe (Lipschitz) values. A plain port, no traps — good to just read:

WGSL 
fn fixDistance(d0: f32, k: f32) -> f32 {
  let sn = sign(d0);
  var d  = abs(d0);
  d = d / k * 1.82;
  d = d + 1.0;
  d = pow(d, 0.5);
  d = d - 1.0;
  d = d * (5.0 / 3.0);
  return d * sn;
}

Your port — pMod2 (the capstone)

This tiles the surface into repeating cells — it's what scatters the little circles across the torus. It is your inout and mod test in one function:

GLSL 
vec2 pMod2(inout vec2 p, vec2 size) {
    vec2 c = floor((p + size*0.5)/size);
    p = mod(p + size*0.5, size) - size*0.5;
    return c;
}

Three things to get right:

  • inout vec2 p → a ptr<function, vec2f> (write through *p);
  • it also returns c — so it both mutates and returns;
  • mod(...) doesn't exist, and % differs from GLSL for negatives — use the glslMod2 helper from the cheat-sheet (or write it: p - s * floor(p / s)).

Post your pMod2 (and glslMod2 if you write it). When it checks out I'll assemble the whole map + renderer and you run the real thing.
