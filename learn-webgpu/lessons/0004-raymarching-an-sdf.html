Lesson 04 / WebGPU for generative art

Into 3D

Raymarching an SDF

Every lesson so far painted a flat plane. Now we go 3D: each pixel fires a ray into a scene and walks it forward until it hits a surface, then shades that hit. You already know all the WGSL here — the new thing is a technique made of four ideas.

1 · The SDF

A signed distance function is the whole scene, expressed as one function: map(p) returns the distance from point p to the nearest surface — positive outside, negative inside, zero on it. A sphere is almost absurdly simple:

WGSL 
fn map(p: vec3f) -> f32 {
  return length(p) - 1.0;   // distance to a sphere of radius 1 at the origin
}

"How far is p from the surface? Its distance from the center, minus the radius." That single number is everything the renderer needs.

2 · The ray

For each pixel we build a ray: an origin ro (the eye) and a unit direction rd pointing through that pixel into the scene.

WGSL 
let pix    = in.uv * 2.0 - 1.0;            // 0..1 uv -> -1..1, centered
let aspect = u.resolution.x / u.resolution.y;
let sp     = vec2f(pix.x * aspect, pix.y); // don't let a wide canvas squash it

let ro = vec3f(0.0, 0.0, 3.0);             // eye, 3 units down +z
let rd = normalize(vec3f(sp, -1.6));       // aim toward -z, through this pixel

3 · The march (sphere tracing — the clever bit)

Here's the insight that makes SDFs magic. If the nearest surface is distance d away, you can safely jump forward by exactly d — you can't possibly overshoot, because nothing is closer than d in any direction. Evaluate, step, repeat:

eye ●─────────▶ . . . . surface
    └─ d₀ ─┘└─ d₁ ─┘└d₂┘▏      each step = the SDF's value at the current point;
                          steps shrink as you near the surface, then you stop.
WGSL 
var t   = 0.0;
var hit = false;
for (var i = 0; i < 96; i = i + 1) {
  let pos = ro + rd * t;
  let d   = map(pos);
  if (d < 0.001) { hit = true; break; }   // close enough = a hit
  t = t + d;                               // jump forward by the safe distance
  if (t > 20.0) { break; }                 // sailed off into empty space
}

4 · Normal & shading

To light the surface you need its normal (which way it faces). The SDF gives it for free: the direction the distance increases fastest is the gradient, found by sampling map a hair in each axis (central differences). Then a dot product with a light direction is basic shading:

WGSL 
let n    = calcNormal(pos);
let light = normalize(vec3f(0.7, 0.8, 0.2));
let diff = dot(n, light) * 0.5 + 0.5;      // Lambert, remapped to 0..1
col = spectrum(diff);                       // ...then tint with YOUR palette

I do — run the engine

All four ideas are assembled, runnable, and heavily commented in src/shader-sketches/wgpu-raymarch.ts:

terminal 
VITE_SKETCH="shader-sketches/wgpu-raymarch" npm run dev

You'll see a shaded sphere, ramped through your spectrum palette. Read the file top to bottom — it's the Lesson 03 sketch plus map, calcNormal, and the march in fs.

Notice the rotation does nothing map already tumbles the whole space with your pR + u.time — but a sphere is perfectly symmetric, so you see no motion. That's the setup for your task.

You do — make it a torus

Port this 3D torus SDF from the original Clifford shader and swap it into map (replace the length(p) - 1.0 line). The instant you do, the pre-wired rotation tilts it and it tumbles.

GLSL 
float fTorus(vec3 p, float smallRadius, float largeRadius) {
    return length(vec2(length(p.xz) - largeRadius, p.y)) - smallRadius;
}

Then call it as fTorus(p, 0.4, 1.0). Nothing exotic in this port — it's a consolidation rep (signatures, vec2f, the .xz swizzle-read). Post your WGSL and I'll grade + wire it.
