Lesson 02 / WebGPU for generative art

Translating, for real

Speaking WGSL

Enough rules. We're going to port three actual functions out of the Clifford Torus shader. I'll translate one with you watching, then you do two and I'll grade them. By the end the dull gradient becomes a rainbow.

The three rules in play

The cheat-sheet has the full table; only three rules matter for today's functions:

  1. Spell out the signature & types. float f(vec3 p) becomes fn f(p: vec3<f32>) -> f32. Drop GLSL's in; a plain parameter is already an input.
  2. Floats are floats. 0 is an i32; write 0.0. vec2(0) becomes vec2<f32>(0.0).
  3. No inout. A parameter the function writes back to becomes either a pointer (ptr<function, T>) or — cleaner — a returned value.

I do — porting smax

Smooth-max, a pure helper. Watch all three rules fire at once:

GLSL 
float smax(float a, float b, float r) {
    vec2 u = max(vec2(r + a, r + b), vec2(0));
    return min(-r, max(a, b)) + length(u);
}
WGSL 
fn smax(a: f32, b: f32, r: f32) -> f32 {
  let u = max(vec2<f32>(r + a, r + b), vec2<f32>(0.0));
  return min(-r, max(a, b)) + length(u);
}

Three changes, nothing more:

  • float … float … → typed params and a -> f32 return.
  • vec2 → vec2<f32>; and vec2(0) → vec2<f32>(0.0) (one scalar splats to both lanes — and it's 0.0, not 0).
  • vec2 u never gets reassigned, so it's a let, not a var. The body maths is identical — min, max, length all exist unchanged.
The point Most of a port is mechanical and boring. The skill is spotting the few spots where a rule bites. That's what the next two functions train.

① The spectrum palette (this one we'll see)

Inigo Quílez's cosine palette. Pure functions — but watch rule 2 hard.

GLSL 
vec3 pal( in float t, in vec3 a, in vec3 b, in vec3 c, in vec3 d ) {
    return a + b*cos( 6.28318*(c*t+d) );
}

vec3 spectrum(float n) {
    return pal( n, vec3(0.5,0.5,0.5), vec3(0.5,0.5,0.5),
                   vec3(1.0,1.0,1.0), vec3(0.0,0.33,0.67) );
}

Hints: in just disappears. cos of a vec3<f32> is fine (per-lane). A scalar times a vector (c*t, b*cos(…)) is fine. Every literal is a float.

② The 2D rotation (the inout one)

This is pR from the torus — it rotates a 2D point in place.

GLSL 
void pR(inout vec2 p, float a) {
    p = cos(a)*p + sin(a)*vec2(p.y, -p.x);
}

You have two valid routes — pick one:
· return-a-value: fn rot2(p: …, a: …) -> … { return …; } (recommended)
· pointer: fn pR(p: ptr<function, …>, a: …) { *p = …; }
Tell me which route and why.
