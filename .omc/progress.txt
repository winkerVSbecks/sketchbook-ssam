## Codebase patterns noted

- ssam Sketch signature: `({ wrap, context, ...props }: SketchProps) => { ... }`
- Sketches register HMR handlers: `import.meta.hot?.on('mcp:export', () => props.exportFrame())`
- Each sketch file calls `ssam(sketch as Sketch<'2d'>, settings)` at the bottom — THAT is the entry point that runs the framework. src/index.ts only side-effect-imports the file.
- This rules out monkey-patching `wrap.render` from index.ts. Will use canvas-existence + 2-rAF settle instead for `__ssam_ready`.
- vite.config.ts /export middleware is HMR-coupled: it requires a live browser page with the handler installed
- Repo has zero WebGL sketches today — Step 6 must author one

## US-001 outcome (Step 0 research, completed)

NO file-based provisioning hook exists in claude.ai/code. Source: code.claude.com docs.
Two mechanisms exist instead:
1. **UI-configured Setup Script** (per-environment, UI only — cannot ship in repo). Runs ONCE per environment before Claude Code launches; result is filesystem-snapshotted for subsequent sessions. Fast for repeat sessions.
2. **SessionStart hook** in .claude/settings.json (repo-committed). Runs on EVERY session start/resume, AFTER Claude Code launches. `CLAUDE_CODE_REMOTE=true` is the cloud-detection env var.

Decision: ship the SessionStart hook as the repo-committed automatic path. Document the UI Setup Script as the recommended faster path. The bootstrap script must short-circuit on cached browser to keep resumed-session latency negligible.
